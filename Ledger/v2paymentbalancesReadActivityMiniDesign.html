Overview

Payment Balances service enables the v2/payment-balance api for both first and third parties to understand sellers' core balance and account concept, with Financial Account primitives as the underlying accounting model interacting with Books, a double entry accounting system.

This doc focuses on the eng design of Payment Balances activity read operation for existing Ledger Books.

Audience

  • Accounting Engineering - Payment Balances team

Terminology

Financial Account primitives

Background & Project Overview

Currently sellers balance and their payment activities are persisted both in Ledger SQL db and in Books, a double entry accounting system. We are migrating the entire accounting system to treat Books as the source of truth to decompose ledger service.

Ledger now directly interacts with Books service. To facilitate interaction with Books and formalize payment and balance models, the end result should be Ledger calling v2/payment-balances API, which calls Financial Account service to interact with Books.

v2/payment-balances API provides READ on account, balance, and balance activities information. In the near future, the v2/payment-balances API will also provide privileged access for internal clients to write to Books through the Financial Account.

The entire project includes 1. enabling READ from v2/payment-balances on existing Ledger Books for a) account, b) balance, and c) balance activities. 2. Reorganize Ledger Books schema to the designed Financial Account schema. 3. Enable privileged parties to write to Books via v2/payment-balances API.

This doc will only focus on part 1c) specifically only on reading Payment Balance activities via v2/payment-balances APIs. Parts 1a) and b) are already complete and live.

Goals

  • Enable READ from v2/payment-balances on Balance Activities.
  • ListPaymentAccountBalanceActivities
  • RetrievePaymentAccountBalanceActivity

Non-Goals

  • Reorganize Ledger Books schema to Financial Account schema
  • Enable privileged WRITE account operation from v2/payment-balances API

Eng Design

Introduction

Now that we have successfully mapped all ledger accounts to Payment Balances, we want to publicly expose being able to read historical activity on these accounts in our connect-v2 APIs. Currently a payment balance activity kafka feed is already mirroring ledger’s journal entry feed. This workstream is solely to expose fetching historical data.
